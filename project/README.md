# Health Insurance Cross-Sell Prediction

## Description

An insurance company wants to identify customers of its **health insurance** product who might also be interested in purchasing **vehicle insurance**. By targeting only those likely to respond positively, the company can optimise its marketing efforts and reduce costs.

This project trains and evaluates machine learning classifiers on a dataset of ~11.5 M labelled customers. Each record describes a customer's demographics, their vehicle details, and their history with the company; the binary label `Response` tells whether they expressed interest in vehicle insurance.

---

## Kaggle Challenge

- **Competition:** [Playground Series, Season 4, Episode 7](https://www.kaggle.com/competitions/playground-series-s4e7)  
- **Task:** Binary classification; predict `Response` (0 = not interested, 1 = interested)  
- **Evaluation metric:** ROC-AUC  
- **Original dataset:** [Health Insurance Cross Sell Prediction](https://www.kaggle.com/datasets/anmolkumar/health-insurance-cross-sell-prediction) by Anmol Kumar

## Notebooks

| Notebook | Purpose |
|---|---|
| `preprocessing.ipynb` | Raw data inspection, type optimisation, and export to Parquet |
| `eda.ipynb` | Exploratory data analysis: distributions, correlations, class balance |
| `model.ipynb` | Feature engineering experiment, model training (Logistic Regression + LightGBM on original vs engineered features), evaluation, and submission export |

> Before running the notebooks, make sure the raw CSV files are present in `data/old/` (see *Data* section below).

## Data

Raw data files (`train.csv`, `test.csv`) should be placed in the `data/old/` directory:

```
project/
└── data/
    ├── old/
    │   ├── train.csv     ← download from Kaggle
    │   └── test.csv      ← download from Kaggle
    └── processed/        ← auto-generated by preprocessing.ipynb
```

Download the files from the Kaggle competition page linked above (login required).  
The processed Parquet files are generated automatically when `preprocessing.ipynb` is run end-to-end.

## Installation and Startup

### 1. Clone or download the project

```bash
git clone <repository-url>
cd MLPW
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows
```

### 3. Install dependencies

```bash
pip install pandas numpy matplotlib seaborn scikit-learn lightgbm pyarrow jupyter
```

### 4. Place the data files

Download `train.csv` and `test.csv` from Kaggle and copy them into `project/data/old/`.

### 5. Run the notebooks

```bash
cd project
jupyter notebook
```

Open and run the notebooks **in order**:
1. `preprocessing.ipynb`: must be run first; it produces the Parquet files consumed by the other notebooks.
2. `eda.ipynb`
3. `model.ipynb`

> **Tip:** Use *Kernel → Restart & Run All* to perform a clean run and ensure cells are numbered consistently from 1.
